Chapter 4
Return Codes

4.1 Introduction

This chapter describes the status and error codes returned by the db.* runtime functions. Table 4-1 lists all the return codes by their numeric value. Section 4.2 is an alphabetical listing of all codes, and includes detailed descriptions of the error messages.

When your program returns an error or status code,

  1. Note the number.
  2. Refer the number to Table 4-1 to get the character code.
  3. Look up the character code in section 4.2, and read a detailed description of the error message.

The descriptions in section 4.2 explain the conditions under which that code would be returned, and, where possible, guidelines are given for resolving the problem. For more information on db.* error handling, see section 5.8, "Database Error Reporting," in the db.* User's Guide.

Note that if you are testing the bounds of error messages within your dberr routine, you can use the following #defines:

  • S_USER_FIRST and S_USER_LAST
  • S_SYSTEM_FIRST and S_SYSTEM_LAST
  • S_FUNCTION_FIRST and S_FUNCTION_LAST

For example, to test for a user error, use the following:

if ((errnum <= S_USER_FIRST) && (errnum >= S_USER_LAST))

4.2 Return Code Descriptions

S_BADFIELD
The field specified is not a field in the current record.

S_BADLOCKS
Inconsistent database lock states have been detected. This error generally occurs when the db.* runtime memory has been corrupted.

S_BADREAD
A read error was returned on a data or key file. Check C's "errno" variable for any additional information.

S_BADTYPE
An invalid access mode argument has been specified in the d_open function. A lock request function returns S_BADTYPE when an invalid lock type is specified.

S_BADUSERID
The user ID contains invalid characters. Since the user ID is usually used to create the log filename, it must contain only alphanumeric characters or underscores ('_').

S_BADWRITE
A write error on database file has occurred. Check C's "errno" variable for additional information.

S_CLOSE
There was an error closing a file. Check C's "errno" for the reason why. This error can indicate that memory has been corrupted.

S_COMKEY
Records containing compound keys cannot be created using function d_makenew, or modified using the field write functions.

S_DBCLOSE
All databases must be closed before calling this function.

S_DBLACCESS
An error has occurred while trying to open the TAF, log, or DBL file, or while trying to get exclusive access to the TAF or DBL file. Be sure that you have read, write, and shareable access to the files and that the application's DBTAF and/or LOCKMGR settings are correct. C's "errno" may be able to give some information about the problem.

S_DBLERR
An error occurred while trying to read or write the TAF or DBL file. Check C's "errno" for the specific error code.

S_DBOPEN
The opened state of the database is not correct for this function. This error code is returned when the database is not opened by functions that must be called after d_open. It is also returned when the database is opened by functions that must be called before d_open.

S_DEBUG
The special debugging checks have been turned on so that the application can be stopped at critical times to test certain features of db.*.

S_DELETED
The record or set has been deleted (since last accessed) by a read function or a timestamp status function.

S_DELSYS
An illegal attempt to delete the system record was made.

S_DUPLICATE
An attempt was made to store a duplicate value into a unique key field. This code is also returned by d_checkid if the proposed user ID is already in use.

S_DUPUSERID
The user ID for this DB_TASK is already being used by another DB_TASK on the system. If you are using a DBUSERID that has been used on another workstation, it may be necessary to run lmclear on the previous workstation to remove the DBUSERID from the name table and other locations.

S_EACCESS
The file is in use, or the file user does not have rights to the file. When possible, under one-user open mode, db.* will open files in a non-shareable state to help the programmer prevent database corruption. This error indicates that the file is opened by db.*, but the file is not under lock manager control.

S_EOS
The end of a set was reached. When a set navigation function or function d_discon has reached the end of the set, S_EOS is returned. Functions d_isowner and d_ismember also return S_EOS when the current record is not currently an owner or member of the set.

S_EXCLUSIVE
Exclusive access is required. The database must be opened in one-user mode or exclusive mode, or exclusive locks must be placed on the appropriate files.

S_FAULT
A page fault has occurred in the db.* cache. Usually this happens with a small cache being shared by multiple DB_TASKs where all the pages are locked; none can be swapped out, but another page is required to be swapped in. This error can also occur when the db.* runtime memory has been corrupted.

S_FSEEK
A seek error was returned on a database file. You can check C's "errno" for the particular error code. If this error occurs during normal operation, it occurred on the log file and could indicate a lack of available disk space. If this error occurs during a recovery, it probably means that the log file has been corrupted. If you get this error during a recovery, you should run dbcheck. If no errors are reported, decide whether you want to continue. (db.* will run smoothly, but dbcheck only detects system-dependent inconsistencies, not application-dependent inconsistencies.)

S_HASMEM
In a call to the d_delete function, the current record is an owner of a non-empty set.

S_INCOMPAT
Incompatible database dictionaries were specified. This code is returned from d_open when the database dictionary for the specified database is not compatible with the version of the db.* runtime that is being executed. You need to run a more recent version of ddlp on your DDL specification, and then retry your d_open call.

S_INVADDR
The database address is invalid. You may have passed the database address variable by value (for example, d_crset(dba)) instead of by reference (for example, d_crset(&dba)). Also, d_recread can return this error if the record slot in the database is corrupted.

S_INVDB
The database name or number is invalid. Function d_open was unable to open the <dbname>.dbd file. You may have incorrectly specified (or not specified) the DBDPATH variable; or you executed from a directory that does not contain the dictionary file. Check C's "errno" variable for additional information. Other functions also return S_INVDB if an incorrect database number is passed to them.

S_INVFLD
The field type is invalid. You may have passed a set or record type to a function that requires a field type. If the current record is of a type other than the one anticipated, S_INVFLD will result, because the field is inappropriate for the current record type.

S_INVID
The d_internals function has been passed an invalid topic or subtopic value.

S_INVLOCK
An invalid lock manager communication type was specified. Make sure the number passed to d_lockcomm, or the string in the INI file, is correct for the types being allowed by the application's LOCKCOMM_DATA table.

S_INVMEM
The record type is not a legal member of the specified set. You may not know what the current record is. Confirm that the previous db.* calls were successful.

S_INVNUM
An invalid record or set type was specified for a locking function. This error code also can be returned from d_setfiles and d_setpages when a number passed in exceeds the limits of the function.

S_INVOWN
The record type is not a legal owner of the specified set. The current record may not be what you think it is. Confirm that the previous db.* calls were successful.

S_INVPTR
A pointer argument to a db.* function is NULL, or a string parameter is the empty string ("") when that is not allowed. This error code can also indicate memory corruption.

S_INVREC
The record type is invalid. You may have passed a set or field type to a function that requires a record type.

S_INVSET
The set type is invalid. You may have passed a record or field type to a function that requires a set type.

S_INVSORT
The specified sort string in a d_mapchar function call is longer than two characters.

S_INVTASK
An invalid DB_TASK is referenced. Be sure that the DB_TASK has been opened and has been passed by reference. Also, be sure that you are not attempting to share the DB_TASKs across applications.

S_ISCOMKEY
Functions that write individual fields cannot be used on fields that make up compound keys. Use d_recwrite instead.

S_ISMEM
In a call to the d_delete function, the current record is still a member of a set occurrence.

S_ISOWNED
In a call to the d_connect function, the current record is already connected to an instance of the specified set.

S_KEYERR
A key file inconsistency has been detected. This usually occurs when either the OS cache or the db.* cache is not being synchronized properly with other processes. This error can also occur when the db.* runtime memory has been corrupted. Run keybuild if dbcheck reveals a corrupted key file.

S_KEYREQD
A key value is required. Function d_makenew returns S_KEYREQD when d_setkey has not previously set all key fields.

S_KEYSEQ
A key sequence error was detected. The function d_keyread was called before a key locate function (for example, d_keynext) had located a key.

S_LMBUSY
The lock manager is busy; and there is no more room in its request queue. Retry the request. Try increasing the number of users (via the -u option) when running the lock manager.

S_LMCERROR
A lock manager communication error has occurred. The error string describes the specific problem.

S_LOCKED
The current record is locked according to the record lock bits.

S_LOGIO
An error has occurred during file I/O on the log file. This error code is used to help distinguish between errors on data and key files, log files, and TAF/DBL files. Check C's "errno" variable for additional information.

S_NAMELEN
The database file or path name is too long. The d_open function returns S_NAMELEN when the 255-character file name limit has been exceeded. The limit is checked after all complete file specifications have been generated from sources such as environment variables.

S_NETERR
A network error occurred. Recovery from this error will depend on your network. A retry of the operation may be all that is necessary. If the lock manager has failed for some reason (for example, if the lock manager host has been re-booted), most active db.* applications will eventually receive this error. The application and the lock manager need to be restarted.

S_NETSYNC
A network synchronization error was detected. An invalid message packet was received from the lock manager. There could have been a transmission error. If only one process has received this error, take it down and bring it back up. If this fails to resolve the problem or if other processes are getting the error, take the entire system down, including the lock manager, and restart. If the error persists, there is probably an error in your network.

S_NOCM
No current member exists. The current member is NULL_DBA.

S_NOCO
No current owner exists. The current owner is NULL_DBA. This can occur when d_findfm is called without first having set the current owner.

S_NOCR
No current record exists. The current record is NULL_DBA.

S_NOFILE
The db.* program is unable to open the specified file. An error occurred when the runtime tried to open a data file, key file, or log file. The db.* program may need more file handles than the current d_setfiles call allows. You may have exceeded the OS limit for file handles. Make sure the data and key files exist.

S_NOLOCKCOMM
No lock manager protocol has been specified. Call the d_lockcomm function or specify a "type" in the [lockmgr] section of the db.star.ini file.

S_NOLOCKMGR
The db.* program was unable to open a lock manager session. The lock manager has probably not been installed.

S_NOMEMORY
Your system is unable to allocate sufficient memory. There is not enough available memory for all of the required db.* runtime tables. Try calling d_setpages with fewer pages before calling d_open.

S_NOPROTOCOL
The chosen communication software has not been loaded, or a path cannot be found. The computer may not be connected to a network, or the network dynamic-link libraries cannot be found.

S_NOSPACE
No more space is available on the disk. See function d_trend in the db.* Reference Manual; a note in the Description explains a technique for dealing with this error under normal operations. However, this error can also occur when the db.* runtime memory has been corrupted.

S_NOTCON
A record is not connected to the specified set.

S_NOTFOUND
A record was not found. The key find function returns S_NOTFOUND when the requested key is not found. The first, last, next, and previous functions return this value when there are no more records to find. The next and previous functions will wrap to the beginning and end of the files, respectively, after this status is returned.

S_NOTFREE
An attempt was made to lock a record, set, or key type that your application had already locked.

S_NOTIMPLEMENTED
A function was called that is not available in this implementation of the db.* library. This may be a debug function that was called in a non-debug implementation of db.*, or it may be a system-specific function.

S_NOTKEY
The specified field used in a key operation is not a key field.

S_NOTLOCKED
An attempt was made to access or free an unlocked record, set, or key type. This may occur when attempting to change a field that is included in a sorted set, and that set is not locked.

S_NOTOPTKEY
A field specified in an optional key operation is not an optional key field.

S_NOTRANS
No transaction has been started. An attempt was made to update the database in shared mode outside a transaction. Database changes can only be made in shared mode from within a transaction.

S_NOTYPE
In a call to d_recnext or d_recprev, a current record type has not been established by a prior call to d_recfrst, d_reclast, or d_recset.

S_NOWORK
No working key file is available. This error generally occurs when the db.* runtime memory has been corrupted.

S_OKAY
A normal return; everything worked successfully.

S_READONLY
Unable to update file due to READONLY option.

S_RECFAIL
Automatic recovery failed. This will be the last of a set of error messages that display. The preceding messages will give more information. For example, an S_BADREAD or S_BADWRITE might be returned before the S_RECFAIL error.

S_RECLIMIT
You have reached the maximum records per file limit for this data file. The first step is to determine if old data can be purged or off-loaded. If so, you will need to write a program to do this. If not, you will need to reorganize your database somewhat. If there are multiple record types in the file, you can split them into separate files (using dbexp and dbimp). You might explore splitting the offending record type into several new record types and dividing the record occurrences based on the content of a principal key field (for example, all names beginning with A-M in record 1 and those with N-Z in record 2).

S_RECOVERY
In a lock request or d_open function call, an automatic recovery operation is about to occur.

S_REENTER
A database task (DB_TASK) was used by two threads simultaneously. Although the db.* library is reentrant, it does not allow reentrant calls with the same database task parameter. Each thread must open its own database task.

S_RENAME
An invalid file number was passed to the function d_renfile.

S_SEM
The db.* program is unable to create a semaphore or mutex.

S_SETCLASH
The internal set pointers changed after you retrieved the record, making further set navigation from this record invalid.

S_SETPAGES
The function d_setpages was called after d_open. The size of the cache cannot be changed once a database has been opened.

S_STATIC
An attempt was made to place a lock on a static record or file. A database must be opened in exclusive (or one-user-only) mode to modify a static record.

S_SYSERR
A system error has occurred. This error generally occurs when the db.* runtime memory has been corrupted.

S_TAFCREATE
The db.* program could not create the transaction activity file (TAF). Check that the TAF name is valid, that the target directory exists, and that the target directory is accessible by your application. The default name of the TAF is db.star.taf, and its path can be specified through a call to d_dbtaf, in the environment variable DBTAF, or in the db.star.ini file. Note that this path should end with a '/' character (on UNIX), or else it will be treated as a file name.

S_TAFLOCK
The db.* program could not access the TAF. This probably means that the TAF was accessed by two applications at the same time, and at least one of those applications opened a database in one-user mode.

S_TAFREOPEN
The db.* program could not open an existing TAF. This may indicate a network error.

S_TAFSYNC
A TAF-lock manager synchronization error has occurred. The TAF tracks the name and type of the lock manager controlling this database family and TAF. To prevent database corruption, the function d_open checks the name and type of the lock manager the application wants to use, against the ones recorded in the TAF to be certain that they match. These values in the TAF can only be changed when the user count in the TAF is zero. The d_open function increments this count, and d_close decrements this count.

S_TIMESTAMP
A record or set is not timestamped. A timestamp function was called for a record or set type that is not being timestamped.

S_TRACTIVE
A transaction was already active. An incremental open cannot be performed during an active transaction. Transactions cannot be nested. That is, you cannot begin a new transaction within an active transaction.

S_TRANSID
A transaction ID was not supplied in a call to d_trbegin or is invalid.

S_TRFREE
An attempt was made to free a lock inside a transaction in shared mode. Only d_trend or d_trabort can free locks once a transaction has begun.

S_TRNOTACT
A transaction was not active. An attempt was made to place a write or keep lock outside of a transaction. Only read locks and exclusive locks can be requested outside of a transaction.

S_UNAVAIL
One or more files that were requested in a lock request function call did not become available before the request timed out, or the database was not available in a call to the d_open function. If the code is returned by d_open, either another user has the database opened in a mode not compatible with your mode, or there is currently not enough room in the lock manager tables to add your user ID and/or your database files.

S_UNLOCKED
The current record is not locked according to the record lock bits.

S_UPDATED
In a call to a timestamp status function, the record or set has been updated since last accessed.

S_USERID
No user ID was specified.
