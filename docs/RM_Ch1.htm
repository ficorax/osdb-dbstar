Chapter 1
Introduction

This book is the reference manual for the Centura db.* product. It provides complete descriptions of all db.* utilities, functions, and return codes. The information provided explains system capabilities in detail, but does not explain how to use them. Information on use of the functions and utilities can be found in the other books of the set.

1.1 db.* Documentation

The db.* documentation is a three-manual set and includes this Reference Manual, the db.* User's Guide, and the db.* Multi-User Guide.

1.1.1 About this Manual

This reference manual is organized as follows:

  • Chapter 2, "Utility Descriptions," contains descriptions of all db.* utility programs.
  • Chapter 3, "Function Descriptions," contains descriptions of the db.* runtime library functions.
  • Chapter 4, "Return Codes," contains descriptions of all db.* status and error codes.

1.1.2 About the Function References in this Manual

For each function described in this reference manual, the reference page header tells you the function name. The first entry on the page is a quick description of the function. This is followed by a Prototypes section that gives a synopsis of the actual function declaration (C-style).

Following the Prototypes section, you will find the Parameters heading, which lists and defines the function parameters introduced in the function declaration.

A Description section follows the parameter definitions. In this section, we explain operational details of the function.

After the function is described, the Currency Changes section provides information on those functions that can cause currency changes.

Following Currency Changes, the function reference presents the Locking Requirements section, which lists locking requirements for any function that accesses the database. If your application has opened the database in a non-shared mode, locking is not required.

The next section is Return Codes and contains the error/status codes for the function, listed in alphabetical order. Detailed definitions of the codes are provided in Chapter 4.

The final sections of the function reference provide an Example of how the function is called and a See Also reference.

1.1.3 Related db.* Documents

This book is the companion volume to the db.* User's Guide and the db.* Multi-User Guide.

The db.* User's Guide describes db.* in detail and provides in-depth instructions for the db.* application developer. The db.* Multi-User Guide contains the information you need to build your application in the multi-user mode.

The db.* system includes a readme.1st file that contains the release notes for your particular operating environment. Be sure to read this file carefully. It provides detailed information about:

  • Recent changes to the system that are not documented in the current manuals
  • Issues specific to your particular operating system or compiler

1.1.4 Document Conventions

Table 1-1 describes the notational conventions used in this manual.

Table 1-1. Document Notational Conventions

Convention

Description

Bold Text

Indicates the names of functions, commands, data structures, statements, databases, utilities, files, etc. Note that the case distinctions used in the names are not significant for all operating systems.

Italic Text

Represents the names of variables, such as parameters and command options. Additionally, the db.* documents use italics for the names of books.

Courier, Colored Text

Indicates a programming example or a command-line entry. Examples are usually enclosed in boxes.

Brackets ([ ])

Indicate optional fields or parameters in command lines and syntax statements.

Bar Symbol (|)

Separates alternative selections. You should enter one of the items separated by the bar.

Ellipsis (...)

Indicates that code has been omitted. Both horizontal and vertical ellipses are used.

Arrow Symbols (< >)

Indicate information that you must supply. For example, <filename>.h represents a header file that you name. Arrow symbols are also used in text to set off keyboard key names, as in <Ctrl>.
